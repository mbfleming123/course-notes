JavaScript Syntax

Syntax

Syntax is a set of rules which define how a computer language should be written.

Comments

Multi-line comments are enclosed with slashes and stars (like in CSS).

/* This comment
   appears on
   three lines. */

Single-line comments are preceded by a double slash.

// Comments may appear at the start of a line or...

var color = "blue";     // ...after a piece of code.

Case Sensitivity

JavaScript is case sensitive.

myCat is not the same as mycat or MYCAT.

Semicolon

A semicolon (;) is used to end each statement.

x = 3 + 2;

Omission of a semicolon may result in an error.

Quotes

Quotes (single or double) distinguish strings from variables, functions, keywords and reserved words.

'Hello!' is a string.

"Hello, world!" is also a string.

helloWorld is not a string. (It may be a variable or function.)

Parentheses

Parentheses indicate an argument to a statement, function or method. They are also used to change the order of operations in arithmetic expressions.

document.write("Hello!");

var solution = ((4 + 76) / 3);

Curly Braces

Curly braces ({ }) group statements into compound statements or "blocks" such as functions or dependent statements.

var x = 2;
if (x == 2) {
  document.write("the variable x holds");
  document.write(x);
};

Period

A period or "dot" (and word following) is used to denote a property or method of an object.

document.write("hello");
// the method "write" is of the object "document"

Reserved words

All programming languages have reserved words which describe built-in functions, objects, properties, methods, etc.

For example: break. else, for, if and while are among the reserved words in JavaScript.

These words cannot be used to declare variables, functions, methods, etc.

Once you are "fluent" in a programming language, these words become familiar. Until then, you may need to refer to resource for a list of reserved words.
