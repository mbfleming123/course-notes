JavaScript 101

JavaScript

  • a high-level, interpreted language
  • the secret sauce of web interactivity
  • introduced in Netscape Navigator 2.0 (1995)
  • commonly used in browsers for client-side scripts
  • has gained popularity for server-side uses (i.e. node.js)

Client-Side vs. Server-Side

Client-Side (a.k.a. front-end) refers to processing handled by a user's computer.

Server-side (a.k.a back-end) refers to processing handled by a web server.

Programming for Interactivity focuses on client-side JavaScript.

JavaScript is not Java!

JavaScript and Java are not the same. They have different syntaxes and uses.

JavaScript is an interpreted language. Java is compiled.

Please don't say we are learning Java ... this is a teeny pet peeve. >:}

Browser Wars

Netscape and Microsoft battled to dominate browser market share during the mid-1990s.

Netscape Navigator and Internet Explorer introduced proprietary HTML tags. They had (similar, but) competing versions of scripting languages, too.

(As of January 2016, Microsoft has ended support for older versions of IE. This makes cross-platform developing a little less of a headache.)

Standard JavaScript

ECMA (European Computer Manufacturers Association), through assistance from Netscape, developed a standard version of JavaScript in 1997.

The standardized version is ECMAScript, but everyone still calls it JavaScript.

The ECMAScript 6 standard was finalized in June 2015.

ECMAScript 6

ECMAScript 6 introduces new features (some which fix JavaScript's 'weirdness').

However, is not yet fully supported across browsers and servers.

  • ECMAScript Compatibility Table

"Programming for Interactivity" will focus on using JavaScript up to version 5.

Considerations

JavaScript is standardized and widely adopted, but some users:

  • may turn off JavaScript functionality
  • surf with older browsers which may lack support
  • lack bandwidth to download large scripts, libraries
  • lack computing power to handle client-side processing
