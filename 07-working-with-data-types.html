Working with Data Types

Operators

Data by itself is useless. We must be able to act on the data to process it.

We can do this in a most basic form using operators

  • assignment
  • arithmetic
  • string
  • comparison
  • logical

Assignment

The assignment operator (=) assigns a value to a variable.

var room = 3032;
room;

Arithmetic

These should be quite familiar:

  • addition (+)
  • subtraction (-)
  • multiplication (*)
  • division (/)

These may not be:

  • modulo (%)
  • increment (++)
  • decrement (--)

Order of Operation

Programming has mathematical roots, so calculations follow Algebra's order of operation (multiplication and division before addition and subtraction).

If you need to change this, enclose the calculation in parentheses ( ( ) ).

20 + 30 * 10     // 320
(20 + 30) * 10   // 500

Modulo

The modulo (%) operator divides two values and returns the remainder.

10 % 3  // 1

(Modulo can be helpful in trying to determine whether something is even or odd or a multiple of X.)

Increment

The increment (++) operator adds one to the current number.

var i = 99;
i++;
i;  // 100

(Increment is often used in loops which count up.)

Decrement

The decrement (--) operator subtracts one from the current number.

var i = 100;
i--;
i;  // 100

(Decrement is often used in loops which count down.)

Arithmetic & Assignment

Arithmetic and assignment operators can be combined, creating a shortcut for modifying values.

  • add and assign (+=)
  • subtract and assign (-=)
  • multiply and assign (*=)
  • divide and assign (/=)
  • modulus and assign (%=)

(These, too, are often used in loops where each step is greater/less than 1.)

Calculate & assign

The long way:

var x = 2;
x = x * 2;
x;

The short way:

var x = 2;
x *= 2;
x;

As you can see, these work the same way; use is a matter of choice.

String

The concatenation (+) operator to combines strings.

"Hello " + "world!"  // Hello, world!

(Uh, didn't we use + to add numbers?)

Strings & Numbers

JavaScript is an untyped language, which means we never have to explicitly say what our data is. JavaScript knows 3 is a number and "3" is a string.

Let's prove this:

typeof 3;    // number!
typeof "3";  // string!

Numbers as strings

If we enclose numbers in quotes, JavaScript will treat them as strings.

"1" + "2"  // "12"

If we concatenate a number with a string, the number becomes part of the string.

2008 + " Florida Gators"  // "2008 Florida Gators"

Try these!

"10" - "5"  // ?!
"10" * "5"  // ?!?!

(Just one example why JavaScript is regarded as quirky.)

Strings are not numbers

If we 'calculate' strings, the result is NaN or "Not a Number."

"ten" - "five"  // NaN

Comparison

Comparison operators are used when our code makes decisions or loops.

When one value is compared to another, the return value is true or false.

  • greater than (>)
  • greater than or equal to (>=)
  • less than (<)
  • less than or equal to (<=)
  • equal to (==)
  • not equal to (!=)
  • strict equal to (===)
  • strict not equal to (!==)

Strict or Not?

Using strict equal to (===) or strict not equal to (!==) compares the value and data type.

"10" === 10    // false
"10" !== "10"  // false
10 === 10      // true

Logical

Logical operators are used when our code makes decisions or loops.

Logical operators compare expressions and return true or false values.

  • and (&&)
  • or (||)
  • not (!)

and

Both expressions/values must evaluate to true to return a true value.

true && true    // true
true && false   // false
false && true   // false
false && false  // false

or

One expression/value must evaluate to true to return a true value.

true || true    // true
true || false   // true
false || true   // true
false || false  // false

not

The opposite value is returned

!true   // false
!false  // true
